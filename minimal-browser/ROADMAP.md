# Roadmap

This document outlines future features and improvements that are not essential for the core browser functionality but would enhance the user experience.

## Short Term (Next 3 Months)

### Essential Features
- [ ] Session persistence to SQLite
- [ ] Tab unload/restore mechanism
- [ ] Basic password manager
- [ ] History storage
- [ ] Bookmarks

### Quality of Life
- [ ] Tab favicon display
- [ ] Tab preview on hover
- [ ] Search in page (Ctrl+F)
- [ ] Print support
- [ ] Download manager
- [ ] Basic developer tools (inspector)

## Medium Term (3-6 Months)

### Advanced Features
- [ ] Tab groups/collections
- [ ] Custom keyboard shortcuts
- [ ] Themes (dark/light/system)
- [ ] Extensions API (basic)
- [ ] Sync across devices (optional, encrypted)
- [ ] Reading mode
- [ ] PDF viewer integration

### Performance
- [ ] Preload optimization
- [ ] Memory compression
- [ ] Disk cache management
- [ ] Process model tuning UI

### Security
- [ ] Certificate pinning
- [ ] Enhanced privacy mode
- [ ] Tracker blocking
- [ ] Content filtering

## Long Term (6+ Months)

### Advanced UI
- [ ] Customizable UI layout
- [ ] Tab tree view
- [ ] Split view (side-by-side tabs)
- [ ] Picture-in-picture
- [ ] Gesture support

### Integration
- [ ] Native notifications
- [ ] System tray integration
- [ ] Wayland protocol support
- [ ] X11 integration improvements
- [ ] Desktop environment integration

### Developer Features
- [ ] Full WebKit inspector
- [ ] Extension development tools
- [ ] Performance profiling UI
- [ ] Network monitor

### Experimental
- [ ] WebAssembly-based extensions
- [ ] Custom rendering engine options
- [ ] VR/AR support (experimental)
- [ ] AI-powered features (privacy-preserving)

## Non-Goals

The following features are explicitly **not** planned:

- Full Chromium/Electron compatibility layer
- Proprietary codec support (use system codecs)
- Cloud sync by default (opt-in only)
- Telemetry or analytics (opt-in only)
- Ad blocking (use system-level solutions or extensions)
- Built-in email client
- Social media integration
- Proprietary DRM (use system DRM if available)

## Community Contributions Welcome

We welcome contributions for any roadmap items. Please open an issue to discuss before starting work on major features.
